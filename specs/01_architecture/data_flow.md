# データフロー仕様

## 1. 記事インジェスションフロー

```
[スケジューラ] ──トリガー──► [RSS ワーカー]
                                   │
                           RSS XML フェッチ
                                   │
                           エントリーパース
                                   │
                    ┌─── 重複チェック ───┐
                    │  (URL の SHA-256)  │
                既存あり           新規記事
                    │                  │
                  スキップ         INSERT articles
                                   (status=pending)
                                         │
                              処理ジョブをキューに投入
                              (article_id → ジョブキュー)
```

**不変条件:** 記事は必ず1回のみ挿入される。重複排除は `articles` テーブルの `url_hash`
ユニーク制約で保証する。

---

## 2. 記事処理フロー

```
[処理ワーカー] ──ポーリング──► [ジョブキュー]
                                    │
                              ジョブ取得 (article_id)
                                    │
                     ┌──────────────▼──────────────┐
                     │  ステージ1: 正規化            │
                     │  - HTML ボイラープレート除去   │
                     │  - クリーンテキスト抽出        │
                     │  - 言語検出                   │
                     └──────────────┬───────────────┘
                                    │
                     ┌──────────────▼──────────────┐
                     │  ステージ2: 埋め込み生成       │
                     │  - 埋め込み API 呼び出し       │
                     │  - article_embeddings に保存  │
                     │    (pgvector)                │
                     └──────────────┬───────────────┘
                                    │
                     ┌──────────────▼──────────────┐
                     │  ステージ3: 要約生成           │
                     │  - LLM API 呼び出し           │
                     │  - article_summaries に保存   │
                     └──────────────┬───────────────┘
                                    │
                     ┌──────────────▼──────────────┐
                     │  ステージ4: タグ付け           │
                     │  - LLM API 呼び出し           │
                     │  - tags テーブルに upsert     │
                     │  - article_tags に関連付け    │
                     └──────────────┬───────────────┘
                                    │
                     ┌──────────────▼──────────────┐
                     │  ステージ5: スコアリング       │
                     │  - トレンドスコア計算          │
                     │  - articles.trend_score 更新  │
                     └──────────────┬───────────────┘
                                    │
                           UPDATE articles
                           SET status='processed'
                                    │
                   レコメンドリフレッシュジョブを投入
                   (対象ユーザーの興味に一致する場合)
```

**ステージ失敗時の処理:**
- 各ステージは `processing_jobs.current_stage` と `last_error` を更新する。
- 失敗ステージは指数バックオフで最大 `max_retries`（デフォルト: 3）回リトライ。
- リトライ上限超過後は `status=failed` とし、アラートを発行する。

## 3. レコメンドフィード生成フロー

```
[レコメンドワーカー] ──トリガー──► (新規記事バッチ完了 または スケジュール)
                                            │
                              ユーザー興味プロフィールを読み込む
                              (user_interests テーブル)
                                            │
                          アクティブプロフィールを持つ各ユーザーに対して:
                                            │
                           ┌────────────────▼───────────────────┐
                           │  候補記事の取得                       │
                           │  - 直近 N 日間に処理済みの記事        │
                           │  - 言語設定でフィルタ                  │
                           └────────────────┬───────────────────┘
                                            │
                           ┌────────────────▼───────────────────┐
                           │  スコアリング                         │
                           │  relevance_score                     │
                           │    (埋め込みベクトルのコサイン類似度)   │
                           │  + freshness_score (公開日の新しさ)   │
                           │  + trend_score (トレンド指数)         │
                           │  + source_quality_score (ソース品質)  │
                           │  + personalization_boost (行動実績)   │
                           └────────────────┬───────────────────┘
                                            │
                           ┌────────────────▼───────────────────┐
                           │  説明生成                             │
                           │  - スコアコンポーネントを人間が読める  │
                           │    理由テキストにマッピング             │
                           └────────────────┬───────────────────┘
                                            │
                             UPSERT recommendation_logs
                             (user_id, article_id, score,
                              explanation, generated_at)
                                            │
                             Redis フィードキャッシュを無効化
                             (対象ユーザー分)
```

---

## 4. ユーザーフィードリクエストフロー

```
[ブラウザ] ──GET /api/v1/recommendations──► [API サーバー]
                                                  │
                                         JWT 検証
                                                  │
                                Redis キャッシュ確認 (user_feed:{user_id})
                                       │                        │
                                 キャッシュ HIT            キャッシュ MISS
                                       │                        │
                                キャッシュ済み           recommendation_logs を
                                レスポンスを返す         articles, sources,
                                                         article_summaries,
                                                         article_tags と JOIN
                                                         WHERE user_id=?
                                                         ORDER BY score DESC
                                                         LIMIT 20 OFFSET ?
                                                                 │
                                                        Redis に書き込み
                                                        TTL = 5 分
                                                                 │
                                                        レスポンス返却
```

---

## 5. ユーザーフィードバックフロー

```
[ブラウザ] ──POST /api/v1/feedback──► [API サーバー]
                                            │
                                  user_feedback に INSERT
                                  (user_id, article_id, type)
                                            │
                              非同期: 該当記事のタグに対応する
                              興味シグナルをインクリメント
                              (user_interests 更新)
                                            │
                              ユーザーのフィードキャッシュを無効化
                                            │
                              200 OK
```

**フィードバック種別:** `like`, `dislike`, `save`, `click`, `hide`

---

## 6. データ保持と削除

| データ | 保持期間 |
|--------|---------|
| 生記事（未処理） | インジェストから 90 日 |
| 処理済み記事 | 無期限 |
| レコメンドログ | 30 日 |
| ユーザーフィードバック | 無期限 |
| 処理ジョブログ | 14 日 |
| Redis フィードキャッシュ | TTL 5 分 |
| Redis レートリミットカウンタ | 1 時間スライディングウィンドウ |
