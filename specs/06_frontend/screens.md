## S-01: ランディングページ

**目的:** 未ログインユーザーにサービスの価値を伝え、登録を促す。

### レイアウト構成

```
[ナビゲーションバー: ロゴ | Trending | ログイン | 登録]

[Hero セクション]
  見出し: "AIが選ぶ、あなたのための技術ニュース"
  サブコピー: "RSS・ブログ・ニュースをAIが要約・ランク付け。
               読む価値のある記事だけを、理由とともにお届けします。"
  CTA ボタン: "無料で始める" → /signup

[フィーチャーセクション]
  - AIによる記事要約
  - パーソナライズフィード
  - 推薦理由の透明性

[トレンドプレビュー（静的）]
  直近のトレンド記事 5 件をカード表示

[フッター: About | Privacy | Terms | GitHub]
```

## S-02: 新規登録

**目的:** メールアドレスとパスワードでアカウントを作成する。

### フォーム仕様

```
表示名（テキスト, 必須）
  placeholder: "例: 山田太郎"
  バリデーション: 1〜50 文字

メールアドレス（email, 必須）
  バリデーション: メール形式

パスワード（password, 必須）
  バリデーション: 8文字以上, 英数字混在
  強度インジケーター表示

[登録するボタン]
  送信中はスピナー表示・ボタン無効化

エラー表示:
  - フィールドレベルのインラインエラー
  - サーバーエラー（メール重複等）はフォーム上部にバナー

登録成功後 → /onboarding へリダイレクト
```

## S-04: パーソナライズフィード

**目的:** ユーザーの興味に合わせた記事フィードを提供するメイン画面。

### レイアウト

```
[ヘッダー: ロゴ | For You | Trending | 検索 | ブックマーク | プロフィール]

[コンテンツエリア]
  [フィルターバー]
    言語切り替え: All / 日本語 / English
    タグフィルタ（複数選択）

  [記事フィード（無限スクロール）]
    記事カード × 20 件
    スクロール末尾でさらに 20 件読み込み
    ローディング: スケルトンカード
```

### 記事カードコンポーネント

```
[ソースロゴ + ソース名] [公開日時（相対表示: "2時間前"）]
[タイトル（リンク）]
[AI要約テキスト（2〜3行で truncate、全文は記事詳細へ）]
[タグバッジ（最大3個）] [...件]
[推薦理由テキスト（灰色・小サイズ）"💡 あなたの Go 興味に一致"]

[フッター: [👍いいね] [🚫非表示] [🔖保存] [外部リンク↗]]

状態:
- is_bookmarked=true → 🔖 塗りつぶし表示
- user_feedback='like' → 👍 ハイライト表示
```

## S-06: 記事詳細

認証不要。SSR（SEO 対応）。

### レイアウト

```
[ヘッダー: 共通ナビ]

[記事ヘッダー]
  タイトル（H1）
  ソース名 + 公開日時 + 著者

[AI 要約セクション]
  "✨ AI 要約" ラベル
  要約テキスト（2〜4文）
  "要約は gpt-4o-mini で生成されました" テキスト（小）

[タグセクション]
  タグバッジ一覧（クリックでフィルタ検索）

[アクションバー（認証済みのみ）]
  [🔖 保存] [👍 いいね] [👎 非表示]

[推薦理由（認証済みのみ）]
  "💡 おすすめ理由: ..." テキスト

[元記事を読む ↗ ボタン]
  メインの CTA。外部 URL を新しいタブで開く。

[関連記事（同じタグの記事 5 件）]
```

## S-08: プロフィール・設定

### セクション構成

```
[基本情報セクション]
  表示名（編集可）
  メールアドレス（表示のみ）
  優先言語（選択: 日本語 / English）
  [保存ボタン]

[興味・関心セクション]
  "あなたの興味タグ" ヘッドライン
  現在のタグ一覧（重み付きプログレスバー）
  [タグを追加] ボタン → タグ検索モーダル
  各タグに [✕ 削除] ボタン
  [保存ボタン]

[危険ゾーン]
  [アカウントを削除] ボタン（確認ダイアログ付き）
```

## S-09: オンボーディング（初回興味設定）

登録直後に一度だけ表示される。

### ステップ構成

```
ステップ 1: "どの分野に興味がありますか？"
  タグカードグリッド（クリックで選択）
  選択済みカードはハイライト
  最低 1 個選択で「次へ」が有効

  [カテゴリ: 言語]    go, rust, python, typescript, ...
  [カテゴリ: インフラ] kubernetes, docker, aws, ...
  [カテゴリ: AI・ML]  llm, machine-learning, ...
  [カテゴリ: トピック] backend, frontend, security, ...

ステップ 2: "優先言語を選んでください"
  ラジオボタン: 日本語 / English / 両方

[スキップ] [完了してフィードを見る]
```

## 共通 UI 仕様

### ナビゲーションバー

```
[Signalix ロゴ] [For You] [Trending] [検索 🔍]
                                     [アバター ▼ → プロフィール / ログアウト]
未ログイン時: [ログイン] [登録]
```

### エラーページ

- `404` → "ページが見つかりません" + ホームへ戻るリンク
- `500` → "サーバーエラーが発生しました" + 再読み込みボタン

### ローディング状態

- 記事フィード → スケルトンカード（3 枚）
- ページ遷移 → Next.js の ProgressBar（トップバー）
- ボタン押下 → スピナー + ボタン無効化

### レスポンシブ対応

- モバイル（〜767px）: 1 カラム。ナビはハンバーガーメニュー。
- タブレット（768px〜1023px）: 1 カラム。サイドバーなし。
- デスクトップ（1024px〜）: メインコンテンツ + サイドバー（任意）。
